# Hermes 微信端基本使用手册

适用环境：Hermes Agent `v0.19.0`，微信 iLink 通道。当前微信端开放的能力为对话澄清、任务看板、长期记忆、图片理解和博客管理。

## 1. 最常用的使用方式

直接在已连接的 Hermes 微信会话中发消息即可。普通任务不需要输入斜杠指令，用自然语言说明目标会更有效。

推荐按下面的顺序描述任务：

```text
目标：你要 Hermes 做什么
材料：正文、图片或已有信息
要求：语气、格式、长度、时间范围
执行方式：直接执行，或者先给预览、等我确认
```

示例：

```text
请把下面这段想法整理成 150 字以内的日常，保留我的语气。
先给我预览，不要发布：……
```

```text
请记住：我写博客时偏好简洁、克制的中文，不使用夸张标题。
```

```text
请把这件事拆成待办，按优先级排列，并告诉我下一步只需要做什么。
```

## 2. 微信对话的正确节奏

- 一条消息尽量只交代一个主要任务；复杂任务可让 Hermes 先列步骤。
- Hermes 正在处理时，不要重复发送同一请求。重复消息可能造成重复执行，尤其是发布内容时。
- 要补充当前任务，用“补充：……”；要改变正在执行的方向，使用 `/steer`。
- 要把另一项工作排到当前任务之后，使用 `/queue`。
- 发送图片后紧接着说明要识别、比较、整理还是发布。多张图片要明确顺序。
- 开始无关的新话题时可使用 `/new`，避免旧上下文干扰。

当前接入主要面向与机器人的私聊。普通微信群消息受微信 iLink 能力限制，不应作为可靠的操作入口。

## 3. 常用微信指令

斜杠指令用于管理 Hermes 会话本身；博客操作通常直接用自然语言。

### 查看与帮助

- `/help`：显示当前可用的常用指令。
- `/commands`：分页浏览全部可用指令；例如 `/commands 2`。
- `/whoami`：查看当前微信身份的指令权限。
- `/status`：查看当前会话、模型、上下文和运行状态。
- `/version`：查看 Hermes 版本。
- `/agents`：查看正在运行的代理或后台任务，别名为 `/tasks`。

### 会话管理

- `/new`：开始一个全新会话；也可用 `/new 文章整理` 创建带名称的新会话。
- `/title 名称`：给当前会话命名，便于以后查找。
- `/sessions`：浏览历史会话。
- `/resume 名称`：恢复一个已命名会话。
- `/retry`：重试上一条请求。
- `/undo`：撤销最近一轮对话记录；`/undo 2` 表示撤销最近两轮。
- `/compress --preview`：先预览上下文压缩效果；长会话接近上限时再使用。
- `/stop`：停止当前正在运行的任务。

`/new` 和 `/undo` 会改变当前会话历史。执行前按微信中的确认提示操作。

### 任务进行中

- `/queue 任务内容`：把任务排到当前任务之后，不打断正在进行的工作。
- `/steer 新要求`：在下一次工具调用后调整当前任务方向。
- `/background 任务内容`：启动一项后台任务，别名为 `/bg` 或 `/btw`。
- `/agents`：查看后台任务状态。

示例：

```text
/queue 当前文章完成后，再检查今天是否有待审核评论
```

```text
/steer 不要改写原句，只修复 Markdown 格式
```

### 长期记忆

当前环境开启了“写入需确认”。你可以先自然语言要求 Hermes 记住偏好，再管理待确认记录：

- `/memory pending`：查看等待确认的记忆。
- `/memory approve <id>`：批准一条记忆。
- `/memory reject <id>`：拒绝一条记忆。

`/new` 只重置当前对话，不会自动删除已批准的长期记忆。

## 4. 图片怎么用

当前微信端已启用图片理解。发送图片后说明目的，例如：

```text
请识别这张截图中的错误信息，只给我原因和下一步。
```

```text
这三张图用于一条日常，按发送顺序排列。先总结你识别到的内容，等我确认后发布。
```

图片理解和博客图片发布是两步：Hermes 先理解图片；当你明确要求发布时，再通过博客管理接口上传并关联到内容。

## 5. 当前能力边界

微信端可以：

- 日常问答、整理、改写、总结和任务规划。
- 理解你在微信中发送的图片。
- 保存经你批准的长期偏好。
- 使用受控的博客管理工具维护文章、日常、书架等内容。

微信端不可以：

- 任意执行服务器 Shell、Docker 命令或直接操作数据库。
- 任意浏览服务器文件系统。
- 默认进行开放互联网浏览。

这些限制是当前部署的安全边界，不是故障。需要服务器维护时，应单独通过受控运维流程处理。

## 6. 安全规则

- 不要在微信中发送密码、API Key、Cookie、证书私钥或服务器密钥。
- 发布、删除、批量修改前，明确写“直接执行”或“先预览，等我确认”。
- 删除属于永久操作，应先让 Hermes 读回目标标题、ID 和摘要，再明确确认删除。
- 大批量修改博客内容前，要求先创建数据库快照。
- 不建议在微信中使用 `/update`、`/restart`、`/model` 或 `/yolo`。这些属于维护或高风险配置操作，尤其不要开启 `/yolo` 跳过审批。

## 7. 故障排查

1. 长时间没有回复：先等待当前任务结束，再发送 `/status`；不要连续重复原请求。
2. 方向不对：发送 `/steer 具体修正要求`。
3. 任务卡住：发送 `/stop`，然后用更短、更明确的请求重试。
4. 博客工具刚更新但未识别：在空闲时发送 `/reload-mcp`。
5. 对话上下文混乱：发送 `/new` 后重新说明目标和必要材料。
6. 微信会话完全无响应：这通常需要服务器侧检查网关，不要使用第二个 Hermes 实例重复登录同一个微信通道。

## 8. 一句话原则

日常工作用自然语言；管理会话才用斜杠指令。涉及公开发布、删除或批量修改时，把“对象、范围、是否直接执行”说清楚。
